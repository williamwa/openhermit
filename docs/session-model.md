# Session Model Docs

Session modeling is also versioned now.

## Documents

- `v1` session model: [docs/v1/session-model.md](v1/session-model.md)
- `v2` architecture and plan: [docs/v2/architecture.md](v2/architecture.md), [docs/v2/plan.md](v2/plan.md)

## Notes

`v1` documents the current session-oriented agent-local API and adapter binding model.

`v2` is expected to keep the session/thread model, but move session persistence, memory, and scheduling responsibilities into a program-level store.
